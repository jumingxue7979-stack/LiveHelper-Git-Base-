# Android Live Rank App

목적: Android 방송자가 라이브 도우미를 방송 시작 입구처럼 쓰면서, 방송자 전용 라이브 순위 팝업을 검증한다.

이 앱은 아직 정식 상품 앱이 아니다.  
1차로 Android PRISM 방송과 YouTube 앱 직접 라이브에서 방송자 전용 순위 팝업이 가능한지 검증하는 APK다.

## 설치 파일

빌드 결과:

```text
dist/LiveHelper_AndroidLiveRank_CompactReport_2026-05-17.apk
```

2026-05-17 모바일 요약표 리포트 APK는 노필터 검색 순위 보정을 유지하고, 방송 시작 후 50분이 지나면 내 채널과 노필터 검색 결과 안의 실시간 라이브 1위 채널을 비교합니다.
리포트는 앱의 `최근 비교 분석 보기` 버튼에서 확인합니다.


기존 Gate 검증 APK:

```text
dist/LiveHelperGate_AndroidPRISM_2026-05-14.apk
```

기존 Gate APK는 방송자 전용 표시 가능성 검증용이고, 새 APK는 실제 라이브 순위 기능 1차 구현용이다.

## 사용 흐름

첫 실행:

```text
API 키
채널 주소
키워드
다른 앱 위에 표시 권한
```

위 항목을 한 번만 저장한다.

다음 방송부터:

```text
라이브 도우미 순위 실행
PRISM으로 방송 시작 또는 YouTube 앱으로 방송 시작
열린 방송 앱에서 라이브 시작
```

앱은 방송 방식 버튼을 누르는 순간 순위 감시를 먼저 켜고, 선택한 방송 앱을 연다.

## 라이브 순위 표시

새 APK는 아래 두 순위를 함께 표시한다.

```text
1. 노필터 순위
2. 라이브 순위
```

표시 방식:

```text
평소: 표시창 없음
방송 시작 직후: 조회하지 않음
방송 시작 5분 후 첫 조회: 큰 팝업 5초
5분마다 새 결과: 큰 팝업 5초
그 뒤: 표시창 완전 제거
```

자동 정지:

```text
내 라이브가 아직 확인되지 않으면 최대 3번만 확인 후 자동 종료
내 라이브가 한 번 확인된 뒤 현재 라이브가 없어지면 자동 종료
```

동시 라이브:

```text
같은 채널에서 현재 라이브가 2개 이상이면 지원하지 않음
동시 라이브 감지 안내 후 자동 종료
처음 고정한 라이브가 끝나도 다른 라이브로 자동 이동하지 않음
```

## 테스트 목표

```text
방송자 폰 화면에는 5초짜리 라이브 순위 팝업이 보인다.
PRISM 시청자 화면에는 표시가 보이지 않는다.
PRISM 녹화본에도 표시가 찍히지 않는다.
```

## 테스트 순서

1. Android 폰에 APK를 설치한다.
2. 앱을 실행한다.
3. `다른 앱 위에 표시 권한 열기`를 누르고 권한을 허용한다.
4. API 키, 채널 주소, 키워드를 입력하고 `설정 저장`을 누른다.
5. `PRISM으로 방송 시작` 또는 `YouTube 앱으로 방송 시작`을 누른다.
6. 열린 방송 앱에서 라이브를 시작한다.
7. 시작 직후에는 순위 팝업이 뜨지 않는지 확인한다.
8. 약 5분 후 방송자 폰 화면에는 5초짜리 노란 순위 팝업이 보이는지 확인한다.
9. 평소에는 표시창이 사라지는지 확인한다.
10. 다른 기기에서 시청자로 들어가 표시창이 보이는지 확인한다.
11. 녹화본에 표시창이 찍히는지 확인한다.
12. 방송 앱에서 라이브를 종료한 뒤 라이브 도우미가 자동 정지하는지 확인한다.
13. 같은 채널에서 라이브를 2개 이상 켠 경우 안내 후 멈추는지 확인한다.

## 판정

통과:

```text
방송자 폰 화면에는 보임
시청자 화면에는 안 보임
녹화본에도 안 찍힘
```

실패:

```text
시청자 화면에 보임
녹화본에 찍힘
PRISM 중 표시창이 사라짐
```
